# Get started

EMX2's design system is **code-first**: every component is defined in code (Vue 3 + Tailwind CSS), and these docs are generated directly from the components themselves — live stories provide interactive demos, and props/events/slots tables are auto-generated from the source. The code is the single source of truth; design files are a reference, not the spec.

## What's in here

- **Foundations** — Design tokens: colors, typography, spacing, radius, elevation, and icons.
- **Components** — Reusable Vue components grouped by purpose. Each page has a live demo, an auto-generated API table, and copyable source from the story file.
- **EMX2** — Components wired to the EMX2 schema and backend (data-aware, not purely presentational).
- **Page templates** — Page-level compositions that combine multiple components into a full layout.
- **Prototypes** — Where we test new compositions and provide examples before they become stable components or land in an app.

Browse or search the sidebar to explore; click a component to see it live with its API.

# Phase F2b — module cols on the containing table (server-side, done right)

Owner-confirmed scope (2026-06-14b). Decision + rationale: `.plan/decisions.md` 2026-06-14(b). Blast radius: `.plan/notes/phase-f2-server-merge-recon.md`. Supersedes the client-merge approach.

Core principle: a column's owner is `col.table`. "Local columns of T" ⇔ `col.table === T.id` (NOT `!inherited`; that was an is-a-only proxy — module cols are `inherited=false`, `table=module`). Module cols surface on the RENDERED table (root OR is-a subclass of it), collected from that table's ROOT, gated (in the form) by `col.table ∈ activeModules`.

## Stage 1 — BACKEND: root-anchor module-subtype resolution + serialize module cols on the table
Module-subtype resolution today anchors on `this` via `getModuleSubtypeTables()` (modules DIRECTLY extending it). A form may render on an is-a subclass of the module-bearing root, so make these ROOT-ANCHORED (modules share the root PK):
1. **Query projection** — `SqlQuery.tableWithInheritanceJoin:901`: left-join `table.getRootTable().getModuleSubtypeTables()` (dedup via existing `joined` set; PK = root PK). ⇒ subclass query projects active module cols. Root unchanged (getRootTable()==self).
2. **Read-by-name** — `TableMetadata.getColumnsIncludingActiveModules`/`getColumnByNameIncludingSubclassesAndModules`/`getColumnsIncludingModules` (:754) + the C4 read surface: resolve modules via root so a module col resolves by name when querying a subclass.
3. **Write routing (C3)** — verify `SqlTable` insert/update module routing (`insertModuleRows`/`expandModuleAncestors`): a subclass insert/update must activate+write the ROOT's modules (discriminator is inherited). Root-anchor if it isn't.
4. **Metadata serialization** — `json/Table.java:71-84`: serialize `getColumnsIncludingModules()` (root-anchored) in addition to the current `getColumns()`, **interleaved by `position`** (stable sort the combined list), **section/heading-safe** (the stateful section/heading loop must NOT be corrupted by module cols — assign module cols their module's section/heading or a default; do the stateful section/heading pass over root cols, then place module cols by position without resetting root state). Each module col already serializes `table=module`, `inherited=false` (json/Column:58,121). Don't change is-a inherited handling.
   - `getColumnsIncludingModules` currently APPENDS unsorted — give the serializer a position-sorted, root-anchored list (new method or sort at the call site).
Constraints: is-a path BYTE-IDENTICAL where there's no module-bearing root (TestDiamondInheritance/TestInherits unchanged). `MODULE` tables themselves: leave their own `_schema` entry as-is.
Tests (red-green, JUnit, embedded PG): subclass query projects active module col / nulls inactive; subclass insert activates module row (+ deactivate removes, C6); `_schema` for ROOT and a SUBCLASS includes module cols with `table=module`,`inherited=false`, position-ordered; root-only/is-a tables byte-identical. Update `TestGraphqlSchemaFields` hard-coded `_schema/.../columns/<index>` paths + `SchemaMetadataGraphqlSurfaceTest` lookups.
